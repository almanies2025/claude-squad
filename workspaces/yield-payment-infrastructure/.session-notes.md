## Architecture Notes

### Threshold Monitoring
- Location: `GET /portfolio` in `main.py`
- Logic: scans ALL historical days per account, finds worst gap, inserts alert if worst > 2.0 bps
- Deduplication: one alert per account per day (checked via `acknowledged = 0` filter)
- Background: runs on-demand when any user calls `GET /portfolio`
- Severity: "warning" if 2.0–4.0 bps, "critical" if > 4.0 bps
- Default threshold: `DEFAULT_GAP_THRESHOLD_BPS = 2.0` (global constant; per-account threshold would require accounts table schema change)

### Security Fixes Applied
- `RateDiscrepancyCreate`: `@field_validator("contract_rate", "applied_rate")` — non-negative only
- `DisputeUpdate`: `@field_validator("notes")` — max 2000 chars
- `update_dispute`: explicit if/elif SQL — no dynamic column interpolation
